Отсоеденённый режим работы с БД

Присоеденённый режим - постоянно открытое подключение программы к бахе данных
каждый запрос образаеться не посредственно к базе данных

Отсоеденённый режим - "импульсное" взаимодействие с БД
- при старте програмы данные загружаются в программу (в массивы\коллекции)
- запросы на выборку данных направлены на загруженные данные 
- в базу данных запросы отправляются либо при закрытии программы,
 либо с некоторой переодичностью для синхронизации данных.


 Отсоеденённый режим значительно быстрее, но может приводить к определённым конфликтам - не актуальности данных, из за редкой синхронизации.
 Этот режим хорош тогда , когда у БД один клиент (одна программа работает с БД) либо когда данные изменяються редко, в основном "на чтение"

 Присоеденённый режим используется в многопользовательсвом режиме или при частных изменениях данных (сайты, месенджеры)

 Для работы с загруженными данными популярна технология MVVM
 Его особенность - связывание представления с данными. ДЛя этого в WPF используються специальные типы, например, ObservableCollection
 Изменения в этой коллекции приводят к появлению внутренних событий которые обновляют представление.
 (! Коллекция должна быть паблик для доступности из представления )
